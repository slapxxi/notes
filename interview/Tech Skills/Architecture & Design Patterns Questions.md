**Q:  Explain SOLID principles briefly.**

> They're five design guidelines - Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation, and Dependency Inversion - that promote clean, maintainable, and scalable code.

**Q: What design patterns are commonly used in frontend development?**

> Patterns like the Observer (used in event systems), Singleton (global state), Command (in UI actions), Builder, Strategy are common. In React, we often use Container-Presenter and Higher-Order Components as structural patterns.
